### SLIDE 2: TECH STACK RATIONALE
**Title:** "Production-Ready Technology Choices"

**Frontend (React 18 + Vite)**
- Why: Fast HMR, modern tooling, component reusability
- Performance: <2s initial load, 98 Lighthouse score
- State Management: Zustand (lightweight, performant)
- UI: Tailwind CSS + Framer Motion (smooth animations)
- Charts: Recharts (responsive, accessible)

**Backend (Node.js 20 + Express)**
- Why: JavaScript full-stack, non-blocking I/O, npm ecosystem
- Database: MongoDB (flexible schema for PII types)
- Caching: Redis (session management, rate limiting)
- Job Queue: Bull (async scan processing)
- Real-time: Socket.io (live scan progress)

**AI Engine (Python FastAPI)**
- Why: Python dominance in NLP/ML, fast async framework
- NLP: spaCy (entity recognition for names)
- PII Detection: Presidio (Microsoft's PII framework)
- OCR: pytesseract (document scanning)
- Regex: Custom Indian PII patterns (Aadhaar, PAN, etc.)

**Infrastructure**
- Docker Compose (local dev & production)
- MongoDB 7 (ACID transactions, aggregation pipeline)
- Redis 7 (sub-millisecond latency)
- Nginx (load balancing, SSL termination)

### SLIDE 4: AI/ML PIPELINE
**Title:** "Intelligent PII Detection Engine"

**Detection Pipeline:**
```
Input Text/File
    ↓
[Preprocessing]
  - Normalize text
  - Remove special chars
  - Tokenization
    ↓
[Regex Matching] (Fast path)
  - Aadhaar: [2-9]\d{3}\s?\d{4}\s?\d{4}
  - PAN: [A-Z]{5}[0-9]{4}[A-Z]
  - Mobile: [6-9]\d{9}
  - Email: standard regex
  - Credit Card: Luhn validation
    ↓
[NER (Named Entity Recognition)] (spaCy)
  - PERSON entities (names)
  - ORG entities (company names)
  - GPE entities (locations)
    ↓
[Risk Scoring]
  - Sensitivity level (sensitive_personal, personal, internal, public)
  - Risk score (0-100)
  - Confidence score
    ↓
[Output]
  - Masked PII (show only last 4 chars)
  - Context snippet (surrounding text)
  - Confidence & risk scores
```

### SLIDE 10: TECHNICAL CHALLENGES & SOLUTIONS
**Title:** "How We Solved Hard Problems"

**Challenge 1: Real-time Scan Progress**
- Problem: Users need live feedback during long scans
- Solution: Socket.io for bidirectional communication
- Result: Live progress updates every 100ms

**Challenge 2: Accurate PII Detection**
- Problem: Regex alone misses context (false positives/negatives)
- Solution: Hybrid approach (regex + NLP + ML)
- Result: 98%+ accuracy

**Challenge 3: Scalable Data Processing**
- Problem: Scanning 1M+ records in reasonable time
- Solution: Async job queues + worker threads
- Result: 10K records in <30 seconds

**Challenge 4: Compliance Audit Trail**
- Problem: Ensuring audit logs can't be tampered with
- Solution: SHA-256 hash chain (each log links to previous)
- Result: Tamper-proof audit trail

**Challenge 5: Multi-Tenancy**
- Problem: Isolating data between organizations
- Solution: orgId in every query, database-level isolation
- Result: Complete data isolation, no cross-org leakage
