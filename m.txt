PRODUCT NAME: DataSentinel
TAGLINE: Know Your Data. Own Your Compliance.
SUBTITLE: India's First Natively DPDPA-Aligned Personal Data Intelligence Platform
CATEGORY: RegTech / Privacy SaaS / Web Security
TARGET USERS: Data Protection Officers, CISOs, Compliance Teams, IT Security Teams in Indian enterprises

UNIQUE ANGLE: Every global tool (BigID, Varonis, Microsoft Purview) was built for GDPR. They don't understand Aadhaar, PAN, UPI, or data written in Tamil, Bengali, or Hindi. They cost ₹1–5 Crore/year. DataSentinel is built from the ground up for Indian law, Indian data, Indian languages, and Indian enterprise budgets.

PROBLEM SOLVED: Organizations store personal data everywhere — mail servers, employee laptops, databases, backups, cloud drives, SaaS tools. Most companies don't know what personal data they hold, where it sits, or how exposed it is. Under the Digital Personal Data Protection Act, 2023 (DPDPA), organizations must identify, classify, and protect personal data. Manual audits don't scale. Spreadsheets won't solve this.

PENALTIES ADDRESSED: DPDPA 2023 allows up to ₹250 Crore per violation. Ignorance is not a defense.

LANGUAGE RULE: JavaScript only. Zero TypeScript. All files .js or .jsx exclusively.
DATABASE RULE: MongoDB with Mongoose. Zero SQL databases.
DATA RULE: Zero seed data. Zero demo data. Zero hardcoded fake data. All data comes from real user actions.

PART 10: BACKEND IMPLEMENTATION DETAILS

server.js:
HTTP server + socket.io init + MongoDB connect + Redis connect + Bull queue init + cron start + listen PORT. Graceful shutdown handling.

app.js:
Express setup → Helmet → express-mongo-sanitize → hpp → CORS (frontend URL) → JSON body → URL-encoded → Winston request logging → Passport.js init → all routes mounted → 404 handler → global error handler last.

encryptionService.js:
encrypt(text) → AES-256-CBC using ENCRYPTION_KEY from env → returns "iv:encrypted" base64 string.
decrypt(encryptedText) → decrypts → returns plaintext.
hashValue(text) → SHA-256 hex (for PII values — never store plain PII).

auditLogger.js middleware:
For every mutating request (POST/PATCH/DELETE) → reads previous audit log entry → computes prevHash → computes entryHash of current action data → inserts AuditLog entry.
Immutable by design — no update or delete routes for audit_logs ever exist.

socketService.js:
socket.io with JWT auth handshake middleware.
Rooms per org: org:{orgId}.
Events: scan:started, scan:progress (every 2s), scan:finding (each PII found), scan:complete, breach:alert, rights:update, policy:triggered.
Bull workers publish to Redis channel → socket.io server subscribes and forwards.

scanService.js + scanWorker.js (Bull):
Bull job processes scan queue.
For each data source: establish connection using decrypted credentials.
Iterate files/records in chunks.
For each chunk: POST to AI engine /detect → receive findings array.
For each finding: hash value (SHA-256), encrypt sourcePath (AES), create PIIFinding document, create/update DataAsset, calculate risk score via riskScoringService, check alert rules via alertService, check policy triggers, emit socket events.
Update ScanJob progress every 100 files.
On complete: store summary in ScanJob.insightSummary.

rightsFulfillmentService.js + rightsWorker.js:
ACCESS request: queries each source for principalId, returns data summary.
ERASURE request: deletes/anonymizes from each source. Records result in fulfillmentLog.
CORRECTION request: updates data in each source.
30-day SLA cron: node-cron checks overdue requests daily, creates alert.

reportService.js:
generateReport(type, params, orgId) → builds full report JSON → uses pdfkit to generate real PDF binary → saves to /uploads/reports/ directory → updates Report record with fileUrl and status "ready".

emailService.js:
nodemailer with SMTP config. HTML templates inline-styled dark theme DataSentinel branded.

Daily 3 AM: check consent records expiring in 30 days → create alerts
Daily 4 AM: check rights requests past 30-day SLA → create alerts

main.py:
FastAPI app. CORS configured for backend origin. Mount all routers. Startup event loads spaCy model + initializes Presidio analyzer.

indian_pii_regex.py — ALL patterns compiled at module load:
AADHAAR: r'\b[2-9]{1}[0-9]{3}\s?[0-9]{4}\s?[0-9]{4}\b'
AADHAAR_MASKED: r'\bXXXX\s?XXXX\s?[0-9]{4}\b'
PAN: r'\b[A-Z]{5}[0-9]{4}[A-Z]{1}\b'
MOBILE: r'(?<!\d)[6-9][0-9]{9}(?!\d)'
VOTER_ID: r'\b[A-Z]{3}[0-9]{7}\b'
PASSPORT: r'\b[A-PR-WYa-pr-wy][1-9]\d\s?\d{4}[1-9]\b'
IFSC: r'\b[A-Z]{4}0[A-Z0-9]{6}\b'
UPI: r'\b[\w\.\-]+@(oksbi|okaxis|okicici|okhdfcbank|ybl|ibl|axl|paytm|apl|upi)\b'
PIN_CODE: r'\b[1-9][0-9]{5}\b' (with context check)
GSTIN: r'\b[0-9]{2}[A-Z]{5}[0-9]{4}[A-Z]{1}[1-9A-Z]{1}Z[0-9A-Z]{1}\b'
CIN: r'\b[LU][0-9]{5}[A-Z]{2}[0-9]{4}[A-Z]{3}[0-9]{6}\b'
TAN: r'\b[A-Z]{4}[0-9]{5}[A-Z]{1}\b'
DRIVING_LICENSE: r'\b[A-Z]{2}[0-9]{2}[- ][0-9]{4}[0-9]{7}\b'
EMAIL: r'\b[A-Za-z0-9._%+\-]+@[A-Za-z0-9.\-]+\.[A-Z|a-z]{2,7}\b'
DOB: r'\b(0?[1-9]|[12][0-9]|3[01])[\/\-\.](0?[1-9]|1[012])[\/\-\.](19|20)\d\d\b'
CREDIT_CARD: Luhn-validated 16-digit groups

pii_detector.py:
detect(text, source_type) function:
Stage 1: Run all regex patterns, collect matches with start/end positions
Stage 2: Run Presidio analyzer on text for global PII
Stage 3: Run spaCy NER for PERSON/ORG/GPE/LOC entities
Stage 4: Merge results, deduplicate overlapping matches (keep highest confidence)
Stage 5: Mask detected values (show only last 4 chars for most)
Return list of Detection objects: {pii_type, masked_value, confidence, start, end, language}

sensitivity_scorer.py:
TYPE_WEIGHTS dict:
AADHAAR=100, BIOMETRIC=100, HEALTH=95, CASTE_RELIGION=95, SEXUAL_ORIENTATION=95, PAN=90, CREDIT_CARD=88, BANK_ACCOUNT=85, FINANCIAL=82, PASSPORT=82, VOTER_ID=78, DRIVING_LICENSE=75, GSTIN=60, MOBILE+NAME_COMBO=72, EMAIL+NAME_COMBO=65, ADDRESS=55, DOB=60, EMAIL=42, MOBILE=52, NAME=35, PIN_CODE=25

Context multipliers (additive, max 100):
unencrypted=+20, internet_exposed=+30, no_consent=+25, retention_expired=+15, bulk_storage=+10, children_data=+20, cross_border_transfer=+15

map_to_sensitivity(score): CRITICAL ≥80, HIGH ≥60, MEDIUM ≥40, LOW below 40
map_to_tier(score): Tier 5 ≥90, Tier 4 ≥70, Tier 3 ≥50, Tier 2 ≥30, Tier 1 below 30

ocr_service.py:
extract_text(file_bytes, mime_type):
If PDF: use Pillow to convert pages to images → pytesseract with language packs (hin, tam, tel, kan, ben, mar, guj for Indic scripts)
If image: pytesseract directly
Return {text, page_count, confidence, extraction_method}

schemas.py (Pydantic models):
DetectRequest: text (str), source_type (str), language_hint (Optional[str])
Detection: pii_type, masked_value, confidence (float 0-1), start (int), end (int), language
DetectResponse: detections (List[Detection]), total_found (int), processing_time_ms (int)
ClassifyRequest: detections (List[Detection]), context (dict)
ClassifyResponse: sensitivity_level, risk_score (int 0-100), tier (int 1-5), recommendations (List[str])
OCRRequest: (multipart file)
OCRResponse: text (str), page_count (int), confidence (float), extraction_method (str)

API endpoints:
POST /detect → DetectRequest → DetectResponse
POST /classify → ClassifyRequest → ClassifyResponse
POST /ocr → multipart file → OCRResponse
POST /batch-detect → List[{id, text}] → keyed DetectResponse results
GET /health → {status: "ok", models_loaded: bool, spacy_loaded: bool}
